Task 1 · Foundation

Project skeleton + date helpers

Create index.html (shell + inline CSS/JS) and test.html (tiny assert harness). Establish the local YYYY-MM-DD date functions everything else depends on — no timezone drift.

Produces: dateKey(d), todayKey(), keyForOffset(n), lastNDays(n).

  1. Write the failing test. Build the assert/assertEqual harness in test.html and add date tests (padding, format, offset 0 = today, lastNDays length/end).
  2. Run to verify it fails. Confirm index.html doesn't yet exist.
  3. Implement. Create index.html skeleton with the four date helpers in its script.
  4. Run to verify it passes. Reload test.html → 6/6 passed.
  5. Commit. feat: habit tracker skeleton + date helpers
Date keys are local, never UTC. dateKey builds the string from getFullYear/Month/Date with zero-padding — using toISOString() would shift days across timezones and corrupt streaks.
Task 2 · Persistence

Store — load / save / migrate

Wrap localStorage with a corruption guard. A bad/missing/old payload returns a fresh default state instead of throwing.

Produces: defaultState(), store.load(), store.save(state), migrate(raw). Key: habit-tracker-v1.

  1. Write the failing test. Use an in-memory backing fake; assert empty/corrupt/old-version all return default, and valid state round-trips.
  2. Run to verify it fails. Confirm store functions absent.
  3. Implement. Add defaultState, migrate, and store (try/catch around JSON.parse) to index.html.
  4. Run to verify it passes. All store tests green.
  5. Commit. feat: localStorage store with migrate + corruption guard
Task 3 · Domain logic

Model operations — pure, immutable

All habit/completion mutations as pure functions: take state, return new state, never mutate the input.

Produces: addHabit, renameHabit, deleteHabit, reorderHabit, toggleCompletion(state,id,key), isDone(state,id,key).

  1. Write the failing test. Add → rename → toggle on/off → delete (asserting completions are dropped too).
  2. Run to verify it fails. First model assertion errors.
  3. Implement. Add the six pure functions (real addHabit uses crypto.randomUUID).
  4. Run to verify it passes. All model tests green.
  5. Commit. feat: pure model operations for habits + completions

Task 7 · UI

History view — 90-day grid + manage

Per-habit screen: a 90-day contribution grid (tap a cell to toggle any day), current + longest streak, and rename / delete (with confirm). Back button returns to Today.

Set currentView before setState. setState calls renderApp, which reads currentView — so cell-toggle/rename set it to History first; delete sets it to Today first.
  1. Define acceptance. Open history; toggle a past cell updates streaks; back; rename; delete drops completions.
  2. Baseline. Click a habit → renderHistory undefined.
  3. Implement. Add renderHistory, extend click handler (back/cellToggle/rename/delete), grid CSS.
  4. Run acceptance steps. All five pass; verify delete via reload.
  5. Commit. feat: per-habit history grid, streak stats, rename/delete
