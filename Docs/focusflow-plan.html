Why

Goals & scope

A distraction-free Pomodoro timer that just works: open the file, hit start, and focus. It must keep accurate time even when the tab is backgrounded, remember your preferences and history across reloads, and run with no server and no network.

In scope
  • Focus / short-break / long-break cycle
  • Configurable durations & rounds-per-long-break
  • Per-day completed-session count + streak
  • Desktop notification + chime at each interval end
  • Installable, offline (PWA)
Out of scope (v1)
  • Accounts / cloud sync
  • Task lists or project tagging
  • Mobile-native app
  • Team / shared timers
Single self-contained index.html (plus a service worker + manifest for the PWA phase). No bundler, no dependencies.
Phase 0 · Foundation

Scaffold & layout — the shell everything renders into

Stand up the single-page shell, design tokens, and the static timer face before any logic.

  1. HTML shell. index.html with a centered timer face, mode label, and a control row (Start / Pause / Reset / Skip).
  2. Design tokens. CSS custom properties for color, spacing, and the big mono time readout; light/dark via prefers-color-scheme.
  3. Responsive layout. Flexbox column that stays centered and legible from phone to desktop.
  4. Module seam. One <script type="module"> with empty stubs (engine, store, ui) so later phases slot in.
Phase 1 · Core

Timer engine — a drift-free interval state machine

The heart of the app: a small state machine that advances through focus and break intervals, and a tick loop that stays accurate even when the tab is throttled.

Interval state machine
Idlestart →
→
Focus25:00
→
Short break5:00 ×3
→
Focus×4 total
→
Long break15:00 → loop
Naïve — drifts
  • setInterval(t--, 1000) decrements a counter
  • Background tabs throttle to ~1/s or pause → lost seconds
  • Sleep / wake desyncs from wall clock
Chosen — timestamp-based
  • Store endsAt = Date.now() + remaining
  • Each tick recomputes remaining = endsAt − Date.now()
  • Display only; truth is the timestamp → survives throttling & sleep
  1. State machine. { mode, round, endsAt, running } with start / pause / reset / skip transitions.
  2. Tick loop. requestAnimationFrame (or 250 ms interval) recomputes remaining from endsAt; format mm:ss.
  3. Auto-advance. On reaching zero, emit an interval-complete event and transition to the next mode.
  4. Pause / resume. On pause, stash remaining; on resume, recompute endsAt.

Phase 5 · Finish

Polish & PWA — shortcuts, themes, installable offline

Round off the experience and make it installable.

  1. Keyboard shortcuts. Space = start/pause, R = reset, S = skip.
  2. Theme toggle. Manual light/dark override on top of the system default; persisted.
  3. Manifest + service worker. Cache the shell for offline; add an install prompt.
  4. Title-bar progress. Favicon ring that fills as the interval elapses.

Proof

Verification

  • Drift test: start a 25-min focus, background the tab 5 min, return — remaining is within ~1 s of wall clock.
  • Cycle test: with short durations, confirm 4 focus blocks → long break, then loop.
  • Persistence: change settings, complete a session, reload — both survive.
  • Streak: seed sessions across days; count + streak match; a gap resets the streak.
  • Offline: install the PWA, go offline, reload — app still loads and runs.
Done = all five checks pass in Chrome + one other engine (Firefox).
